How to Flexbox

Hello and welcome to my first tutorial; this one concerning flexboxes! Code snippets and demos will be provided along the way! Please feel free to copy them and use them on your own projects! If you need any more help after this, please let me know!

I'll keep things as succinct but detailed as possible, starting with...

What is Flexbox?

Flexbox is an extremely dynamic and responsive display type designed around objects within it stretching and shrinking based off of "flexibility" values. Layouts designed with it are extremely easy to adapt to mobile and typically respond well to all laptop and PC viewports (screen sizes).

It is very widely available nowadays, and once you learn it, I'd argue it's easier to use than every other display type. If you look around my site, you will find that I use it just about everywhere, and that is because of just how flexible it is!

To flex a box

That's basically all you need to know concerning what it is. Now, let's get started with basic syntax.


.flex { 
    display: flex;
}

This, of course, denotes whatever object we are flex displaying as a flex box. But what does it do?

Essentially, whatever object we decide to apply this class to becomes a "flex-parent", and everything inside of it become flex-children. The helper modifiers we apply to the flex-parent will affect everything inside of itself only. That becomes important later. Now, let's see a visual of what it does.


.flex {
    display: flex;
}

.parentDiv {
    background-color: var(--deepsea);
    }

.childDiv {
    margin: 10px;
    padding: 20px;
    font-size: 20px;
    background-color: var(--whispermist);
}

I applied some styling to these divs so we can see them for now. I will remove the height and aspect ratio modifiers later, because we can handle all of it with flexbox! So now, if you remove the flex modifer, you'll see that the width of the starter div will grow quite large. What gives?

Well, let me apply some more modifiers to show you the default styling of a flex-parent.


.flex {
    display: flex;
    flex-flow: row nowrap;
    justify-content: flex-start;
    align-items: stretch;
    align-content: stretch;
}

.parentDiv {
    background-color: var(--deepsea);
    }

.childDiv {
    margin: 10px;
    padding: 20px;
    font-size: 20px;
    background-color: var(--whispermist);
}

We added 4 new modifiers. Here's what they do.

flex-flow:

flex-flow is the shorthand of flex-direction and flex-wrap. I typically use flex-flow because I don't see much reason not too, but for possibly a little less typing, you can just use the regular modifiers instead.

flex-flow essentially tells the flex-parent how it should display it's flex-children. There are 8 different aptly-named options for flex-flow, as follows:

row / column nowrap

row and column denote the direction that the code should flow in.

Importantly, this direction can be kind of confusing sometimes, so let's break it down. Essentially, there are 2 main axes in a flex-parent, known as the main axis and the cross axis. In a basic flex-layout, the main axis is the horizontal x-axis, and the cross axis is the vertical y-axis. However, when you choose column as your flex-direction, this flips around because everything is oriented vertically. This becomes important because a lot of the helper modifiers below are modified by this change in the main axis. We will address a lot of this later on, so don't worry about it for now, but it is something to think about when building layouts!

nowrap means that the flex-children won't wrap based on the flex-parent's respective width or height. Experiment with these modifiers below! To see the effect, add a few more starterDivs to the parent.

row / column wrap

nowrap means that the flex-children will wrap based on the flex-parent's respective width or height. Experiment with these modifiers too!

row-reverse, column-reverse, and wrap-reverse

row-reverse and column-reverse can be used to reverse the main-axis direction. wrap-reverse reverses the wrap order too. It's extremely weird IMO. Feel free to experiment with all of this too, but I don't use it almost at all.

Justify Content

justify-content is responsible for positioning flex-children on the main-axis of the flex-parent (whichever one that might be). There are 6 different aptly-named options for justify-content, as follows:

flex-start and flex-end

flex-start and flex-end tell the parent to justify everything to either end of the flex-parent's main axis.
To test the column flex-end, give the parentDiv a bigger height, like 200px or something.

center

center tells the parent to justify everything to the center of the main axis. You know how to test column by now.

space-around, space-between, and space-evenly

space-around displays the flex-children with a space between and around them. They will not look equal visually because each flex-child will have it's own space that gets added to the adjacent flex-children. It's like having 3 divs with the same amount of margin all next to each other.

space-between displays the flex-children with equal space between them.

space-evenly displays the flex-children with completely equal space between and around them.

align-items

align-items is responsible for positioning flex-children on the cross axis of the flex-parent.

This can be overwritten by flex-child properties later, while justify-content cannot be.

stretch

stretch means that the flex-children will stretch to fill up the entire flex-parent (with respect to min and max width and height values.)
